**💡 기획자 코멘트:**

현재 AppSheet 기반에서는 모든 사업부 대시보드가 좌측 메뉴에 동일한 레벨로 나열되어 있습니다. 웹앱으로 고도화할 경우, '통합 대시보드'라는 상위 메뉴를 두고 로그인한 사용자의 권한(IT사업부 담당자, 마이메일러 담당자 등)에 따라 **자신의 소속 사업부 대시보드가 첫 화면(Home)으로 랜딩되도록 설계**하는 것을 추천합니다.

마이메일러 대시보드 분석 데이터를 바탕으로, 개발 기획서의 두 번째 단계인 \[2) 화면별 상세 기능 및 UI/UX 요구사항\]을 작성해 드립니다.

기존 AppSheet의 정적인 화면 구성을 넘어, 웹 앱 환경에서 구현될 **인터랙티브 기능**과 **사용자 편의성**에 초점을 맞추었습니다.

IT사업부 대시보드의 데이터를 기반으로, 각 **모듈(Module)별 세부 화면** 기획안을 정리해 드립니다.

사용자가 언급하신 '레모드'를 '리포트 모드' 또는 '세부 모듈'로 해석하여, 대시보드 내에서 각 지표를 클릭했을 때 진입하게 될 **상세 기능 페이지**의 구성을 설계했습니다.

## **1-2 IT사업부 대시보드 세부 화면 상세 기획**

### **1-2-1. \[모듈 1\] 실적 요약 및 KPI 보드 (Summary Mode)**

메인 접속 시 가장 먼저 보이는 화면으로, 현재 사업부의 건강 상태를 한눈에 파악합니다.

* **주요 UI 요소:**  
  * **KPI 카드:** 전월 수금액, 당월 예정수금, 금주 예정수금, 익월 예정수금을 큰 텍스트로 배치.  
  * **달성률 게이지:** (당월 수금액 / 예정수금)을 퍼센트로 시각화.  
* **상세 기능:**  
  * **실시간 업데이트:** 데이터 변경 시 즉각적인 수치 반영.  
  * **사업부 스위처:** 상단에서 IT사업부/마이메일러 등 사업부 간 빠른 전환 버튼 제공.

### **4-2. \[모듈 2\] 월별 수금 상세 현황 (Monthly Drill-down Mode)**

'전월', '당월', '익월' 버튼을 클릭했을 때 나타나는 상세 리스트 화면입니다.

| 항목 | 상세 기능 요구사항 | 비고 |
| :---- | :---- | :---- |
| **데이터 그리드** | • 업체명, 항목, 수금액, 예정일, 담당자 필드 포함 • 정렬(Sorting) 및 다중 필터 기능 제공 | 엑셀 스타일의 그리드 |
| **상태 관리** | • 수금 완료/미정/지연 상태를 컬러 태그로 구분 • 한 번의 클릭으로 상태 변경(Toggle) 기능 | 운영 효율성 증대 |
| **요약 통계** | • 현재 보고 있는 월의 총 합계액 및 항목별 비중(차트) 표시 | 하단 플로팅 바 배치 |

### **1-2-3. \[모듈 3\] 주차별 매출 추이 분석 (Weekly Trend Mode)**

현재 화면 하단에 위치한 '수금 내역' 데이터를 시각적으로 확장한 모듈입니다.

* **주요 UI 요소:**  
  * **멀티 라인 차트:** 1주차\~5주차까지의 매출액과 수금액을 겹쳐서 표시하여 괴리율 확인.  
  * **특이점 알림:** 전주 대비 매출이 20% 이상 급변한 주차에 마커(Marker) 표시.  
* **상세 기능:**  
  * **기간 선택 도구:** 1분기, 상반기, 특정 월 범위를 자유롭게 선택하여 조회.

### **1-2-4. \[모듈 4\] 수금 미정 집중 관리 (Unresolved Tracker)**

가장 관리가 시급한 '수금 미정' 항목만 별도로 모아놓은 특수 모드입니다.

* **상세 기능:**  
  * **푸시 알림 예약:** 예정일이 지났음에도 미정 상태인 건에 대해 담당자에게 알림 발송 기능.  
  * **메모/히스토리:** 수금이 늦어지는 사유(예: 고객사 검수 지연 등)를 기록하는 히스토리 피드.  
  * **일괄 수정:** 여러 건의 수금 예정일을 한꺼번에 변경하는 벌크(Bulk) 편집 기능.

### **💡 개발 가이드 요약 (UI/UX 포인트)**

**"클릭은 최소화, 정보는 최대화"**

1. **Tab 구조 활용:** 화면 상단에 \[요약 / 월별상세 / 추이분석 / 미정관리\] 탭을 두어 화면 전환 없이 이동하게 구성합니다.  
2. **색상 시스템 정의:** \> \* **Blue:** 예정수금 (안정)  
   * **Red:** 수금 지연/미정 (주의)  
   * **Green:** 수금 완료 (성공)  
3. **반응형 대응:** 데스크탑에서는 넓은 표를 보여주고, 태블릿/모바일에서는 핵심 KPI 카드 위주로 레이아웃이 자동 재배치되도록 설계합니다.

---

대시보드의 시각화된 지표에서 상세 내역으로 바로 접근할 수 있는 **'드릴 다운(Drill-down)'** 기능을 기획안에 추가하겠습니다.

사용자가 차트의 막대나 요약 카드의 버튼을 클릭했을 때, 해당 데이터의 근거가 되는 \[로우 데이터 테이블(Raw Data Table)\]로 즉시 이동하여 세부 정보를 확인하고 수정할 수 있는 동선을 설계했습니다.

## **1-3. \[신규 모듈\] 로우 데이터 테이블 뷰 (Raw Data Detail View)**

지표 클릭 시 전환되는 화면으로, 필터링된 전체 상세 레코드를 보여주는 그리드 형태의 화면입니다.

### **🛠️ 주요 기능 및 인터랙션 (Interaction)**

* **지능형 필터링 연동:** \* 예: 'IT사업부 당월 예정수금' 막대 클릭 시 → '사업부=IT', '수금예정일=당월', '상태=미납' 조건이 **자동 적용된** 테이블로 이동.  
* **실시간 데이터 수정 (Editable Grid):** \* 테이블 내에서 직접 비고란을 입력하거나 수금 상태를 '완료'로 변경 가능.  
* **Excel/CSV 내보내기:** \* 현재 화면에 필터링된 로우 데이터를 즉시 엑셀 파일로 추출하는 상단 버튼 배치.  
* **대시보드 복귀 (Back to Dash):** \* 상단에 '대시보드로 돌아가기' 브레드크럼(Breadcrumb) 또는 플로팅 버튼 제공.

### **📋 테이블 구성 항목 (Column Definition)**

현재 AppSheet에서 확인되는 주요 컬럼들을 웹 앱 환경에 맞게 재구성합니다.

| 컬럼명 | 타입 | 특징 |
| :---- | :---- | :---- |
| **고객명** | 텍스트 | 클릭 시 고객사 히스토리 팝업 오픈 |
| **매출금액** | 수치 (KRW) | 1,000단위 콤마 및 통화 표시 |
| **수금예정일** | 날짜 | 경과 시 붉은색 강조 처리 |
| **진행상태** | 드롭다운 | 계약진행 / 입금완료 / 드랍 등 상태 변경 가능 |
| **비고(Memo)** | 텍스트 영역 | 마우스 오버 시 전체 내용 툴팁 표시 |
| **식별 ID** | 텍스트 | 시스템 관리용 고유 번호 (조회 전용) |

### **💡 사용자 시나리오 (User Journey)**

1. **발견:** 관리자가 IT사업부 대시보드에서 '4월 매출 하락' 그래프를 확인.  
2. **클릭:** 하락 지점의 그래프 막대 또는 하단의 **\[상세 보기\]** 버튼 클릭.  
3. **분석:** 4월의 모든 로우 데이터가 담긴 테이블 뷰로 이동. 필터링된 리스트에서 미납된 특정 업체(예: 충북보건과학대학교) 발견.  
4. **조치:** 테이블 내 '비고'란에 "5월로 수금 연기 확인" 기재 후 저장.

**기획자 의견:** 이 기능은 단순히 데이터를 보여주는 것을 넘어, 현장의 담당자가 '분석 후 즉시 조치'를 할 수 있게 만드는 핵심 기능입니다.

마이메일러 사업부의 매출 특성(신규/재구매/서비스별 분류)을 직관적으로 관리하는 화면입니다.

### **\[화면 B\] 통합 수금 및 매출 관리 (전사 공통)**

모든 사업부의 실적을 한눈에 비교하고 관리하는 컨트롤 타워 화면입니다.

* **멀티 사업부 필터링:** 상단 탭 또는 드롭다운을 통해 IT사업부, 마이메일러, 처음서비스, 프링고 데이터를 즉시 스위칭.  
* **실시간 KPI 위젯:** 당월 총 매출액, 수금 달성률, 미수금 총액을 최상단에 상시 노출.  
* **주간별 트렌드 라인 차트:** 1주차\~5주차 흐름을 선 그래프로 표시하여 '매출 하락 구간' 조기 감지.

### **\[화면 C\] 수금 미정 내역 관리**

운영 효율을 극대화하기 위한 데이터 관리 그리드 화면입니다.

* **인라인 에디팅 (Inline Editing):** 엑셀처럼 화면에서 즉시 수금 예정일이나 상태값을 수정하고 저장.  
* **조건부 서식 (Conditional Formatting):** 입금 예정일이 지난 미정 내역은 행 전체를 붉은색으로 강조.  
* **데이터 내보내기:** 현재 필터링된 미정 내역을 엑셀(XLSX) 또는 PDF로 다운로드하는 기능.

### **🎨 UI/UX 고도화 가이드 (AppSheet 대비 개선점)**

**1\. 반응형 레이아웃 (Responsive Design)**

* 기존의 좁은 모바일 형태에서 벗어나, **데스크탑 와이드 모니터에 최적화된 3열 그리드 레이아웃**을 적용하여 한 화면에서 더 많은 데이터를 탐색할 수 있게 합니다.

**2\. 인터랙티브 데이터 탐색**

* 그래프의 특정 영역을 드래그하여 \*\*특정 기간만 확대(Zoom-in)\*\*해서 보거나, 범례(Legend)를 클릭하여 특정 항목을 숨기는 기능을 추가합니다.

**3\. 다크 모드 및 가독성**

* 데이터 집중도를 높이기 위해 기본 \*\*다크 테마(Dark Theme)\*\*를 제공하되, 고대비 폰트(Pretendard 등)를 사용하여 수치 가독성을 최우선으로 합니다.

나머지 사업부(처음서비스, 프링고)의 UI 구성이 동일하다는 점을 확인했습니다. 

이는 개발 관점에서 컴포넌트의 재사용성(Reusability)을 높여 개발 공수를 줄이고 사용자에게 일관된 경험을 제공할 수 있는 큰 장점입니다.

이어서 개발 기획서의 핵심인 \[3) 데이터베이스 구조(DB Schema) 및 데이터 연동 전략\]를 작성해 드립니다. 기존 AppSheet가 구글 스프레드시트를 기반으로 했다면, 새로운 웹 앱은 대규모 데이터 처리와 복잡한 쿼리에 최적화된 구조를 갖추게 됩니다.

## **3\) 데이터베이스 구조(DB Schema) 및 데이터 연동 전략**

### **🗄️ 3-1. 핵심 데이터베이스 스키마 (ERD 개념)**

데이터 간의 관계를 명확히 하여 사업부별 실적을 다각도로 조회할 수 있도록 설계합니다.

| 테이블명 | 주요 컬럼 (Field) | 설명 |
| :---- | :---- | :---- |
| **BusinessUnits** | unit\_id(PK), unit\_name, manager\_id | 사업부 정보 (IT, 마이메일러, 처음서비스, 프링고) |
| **SalesData** | sale\_id(PK), unit\_id(FK), date, amount, vat, category\_id, customer\_type | 매출 상세 내역 (날짜, 금액, 신규/재구매 여부 등) |
| **Categories** | category\_id(PK), unit\_id(FK), category\_name | 사업부별 서비스 카테고리 (ASP, 개발서비스 등) |
| **Collections** | collection\_id(PK), sale\_id(FK), status, expected\_date, actual\_date | 수금 관리 (예정일, 실제 입금일, 미정/완료 상태) |
| **Users** | user\_id(PK), email, role, unit\_id(FK) | 사용자 권한 관리 (관리자, 사업부별 담당자) |

---

### **🔄 3-2. 데이터 연동 및 전환 전략 (Data Migration)**

기존 구글 스프레드시트 환경에서 전문 DB(PostgreSQL) 환경으로 안전하게 전환하기 위한 전략입니다.

1. **초기 데이터 이관 (EtL):**  
   * 현재 AppSheet에서 사용 중인 구글 스프레드시트 데이터를 정제하여 PostgreSQL로 일괄 마이그레이션합니다.  
   * 중복 데이터 제거 및 날짜 형식 표준화 작업을 선행합니다.  
2. **데이터 동기화 (Hybrid Approach):**  
   * **과기:** 과도기적으로 기존 스프레드시트를 계속 사용해야 할 경우, Google Sheets API를 사용하여 스프레드시트에 값이 입력되면 실시간으로 DB에 반영되도록 동기화 파이프라인을 구축합니다.  
   * **완성기:** 웹 앱 내의 직접 입력 폼을 통해 DB에 바로 저장하고, 필요 시 역으로 스프레드시트에 백업하는 방식을 채택합니다.  
3. **실시간 집계 엔진:**  
   * 대시보드 로딩 속도를 높이기 위해, 매 접속 시 전체 데이터를 계산하는 대신 **'일일 단위 배치(Batch) 처리'** 또는 'Materialized View'를 사용하여 미리 계산된 수치(KPI)를 즉시 불러오도록 설계합니다.

---

### **🚀 3-3. 기술적 차별화 포인트 (AppSheet 대비)**

* **복잡한 연산 처리:** 수천 행의 데이터를 브라우저가 아닌 서버(Node.js)에서 처리하여 대시보드 로딩 속도를 1초 미만으로 단축합니다.  
* **히스토리 추적:** 데이터가 수정될 때마다 '누가, 언제, 어떤 값'을 바꿨는지 기록하는 로그(Audit Log) 기능을 추가하여 수금 데이터의 신뢰성을 확보합니다.  
* **확장성:** 추후 새로운 사업부가 추가되더라도 DB 테이블의 행 추가만으로 즉시 대시보드를 생성할 수 있는 유연한 구조를 유지합니다.
